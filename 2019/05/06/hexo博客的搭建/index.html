hexo博客的搭建

发表于 2019-05-06 | 分类于 hexo博客的搭建

第一步，先安装好node还有git

第二步，在硬盘或者桌面新建一个文件夹，起个名字，我的是hexo名字

第三步，进入 hexo官网：https://hexo.io/zh-cn/

第四步，在hexo官网首页中复制：npm install hexo-cli -g。
然后在hexo那个文件夹右键Git Bash Code 输入/执行后需要等待一段时间

第五步，输入hexo init 执行后需要等待(很长)一段时间 ，
输入npm -v查看版本

第六步，输入hexo s 然后复制http://localhost:4000在浏览器打开，
就可以看到你的hexo主页了。如果再想输入代码的话，
可以ctrl加c停止当前的代码，就可以输入代码了

第七步，下载next主题，在浏览器打开http://theme-next.iissnan.com/这个网址，
点击开始使用，点击下载稳定版本，
点击发布页面，找到Assets下面的Source code(zip)点击一下，就自动下包了。
下载完后把那个压缩包的文件拉到themes文件下，再改名字，改为next

第八步，修改主题，在hexo文件的根目录下 找到_config.yml文件用代码编辑器打开该文件，
ctrl+F 搜索 theme: 把这里的landscape 改成 next，
然后可以在在hexo的根目录下打开Git Bash执行hexo s代码，
可以看到主题已经改了

第九步，修改语言为中文，在hexo文件的根目录下 找到_config.yml文件用代码编辑器打开该文件，
找到 language，然后复制这一段替换language: zh-Hans，
刚替换的时候，语言还替换不了的，在hexo文件的根目录下右键Git Bash，
第一步输入hexo clear 清理缓存，
第二步hexo g 生成文件，
第三步hexo s 启用服务器，
在我们需要打开hexo页面的时候 该文件不能关掉

第十步，开启hexo的标签导航。
打开next文件根目录下的_config.yml，
ctrl+F 搜索menu，把tags，
categories前面加#的符号都去掉 再回到hexo刷新页面，

第十一步，创建标签页面。
在hexo文件的根目录下右键Git Bash，ctrl+c先把hexo服务器关闭，
创建标签页: hexo n page tags，等待完成，会在 source目录下多出一个文件: tags，
打开该文件里的index.md文件，
title: 这个页面的标题名字，我的这个页面是标签页面，所以标题名称是标签，
date: 时间，
type: “tags” 这个必须要填 不然不会显示
创建分类页面，hexo n page categories，等待完成，会在 source目录下多出一个文件: categories，
打开该文件里的index.md文件，
title: 这个页面的标题名字，我的这个页面是分类页面，所以标题名称是分类，
date: 时间，
type: “categories” 这个必须要填 不然不会显示，
然后重新开启服务器，然后点击标签页，就可以看到效果了

第十二步，修改主题，在next的官方文档中有说明，
在next的根目录下打开配置文件 _config.yml，
ctrl+F 搜索关键字 Scheme，然后找到Scheme Settings，
主题都在下面，喜欢那个主题就把前面的#号去掉就可以了，
Muse - 默认 Scheme，这是 NexT 最初的版本，黑白主调，大量留白，
Mist - Muse ，整洁有序的单栏外观，
Pisces - 双栏，版心会比较小，
我用的是scheme: Gemini这个主题，和Pisces一样，就是版心比较大

第十三步，设置头像。
在next的根目录下 打开_config.yml配置文件，
ctrl+F 搜索关键字 avatar，
把avatar: 前面的#号去掉
这里的images是存放图片的 你可以自己存图片进来 images在next的source目录下

第十四步，设置网页标题，
在hexo文件的根目录下 找到_config.yml文件用代码编辑器打开该文件，
然后修改title: 网站的名称，随意你改名字，最好有代表性的一个单词
author: 作者姓名，
url: http://这里填的是title的标题名字.com，
修改完后 刷新hexo页面是无法看到改变的 需要生成文件，
在hexo文件的根目录下右键Git Bash，
第一步输入hexo clear 清理缓存，
第二步hexo g 生成文件，
第三步hexo s 启用服务器，

第十五步，设置 侧边栏社交链接，
打开next文件中的_config.yml配置文件，
搜索关键字: social，把social: 前面的#去掉，
把Github和E-Mail的#号去掉，
然后在github那一行把yourname改成你的标题名称，
再去email这行把email改成weibo，
把github上面的地址复制下来，
然后把github改成weibo，最后面那个后缀也改成weibo

第十六步，启用友情链接，
打开next文件中的_config.yml配置文件，
搜索： #links_layout: inline，把下面那行的links: 前面的#去掉，
Title: http:…. 也把友情链接的#也去掉，Title可以改成你的想要的名字，
这里的链接模仿的是馨客栈的地址，把中间的example改成mackxin，
可以依照第一个来设置多个友情链接，
那个标题名字可以改成中文的，
第二个链接也是模仿馨客栈的，馨客栈导航: http://mackxin.com/nav.html

第十七步，新建一篇文章，
先ctrl+c关闭hexo服务器，
创建文章: 在hexo的根目录下 打开git 输入命令行 hexo n 文章名字 然后回车，
然后 hexo s 开启服务器，然后刷新页面 就可以在首页看到多出来一篇文章

第十八步，让新建的文章在标签和分类页中显示出来，
在hexo根目录中打开source文件夹，打开_posts文件夹，
用代码编辑器打开你刚刚新建的文件，
在tags:回车一行然后TRB空格一下再添加- 导航，再换行再添加- 分享，这两个东西是是标签页显示的东西，
作用是让你可以通过标签来找到那一篇文章，
有时候你删除了一个标签，但是页面没有更新，你就要清除一下缓存那三个步骤。
categories: 前端，这个前端是在分类页面显示出来的文字，
这是一个分类，写正文的话需要在这三个—横杠下面写才有效，

#一级标题， ## 二级标题

第二十步，修改主页的文章的显示，显示一个查看全文的按钮，打开next的配置文件，然后ctrl + f 搜索 auto_
搜到auto_excerpt: 然后把他的enable:false 的值改成 true

第二十一步，修改链接的颜色，首先在hexo根目录中打开themes里面的next里面的source里面的css里面的_common里面的components里面的post文件夹里面的post.styl文件，然后在最下面粘贴以下代码
.post-body p a {
color: hotpink;
border-bottom: none;
&:hover {
color: orange;
}
}

第二十二步，把项目部署到github，实现在线观看。
（1）首先建一个github账号，名字要和你的hexo名称一样，如果名称被注册了，就换一个，然后要知道你的邮箱账号。
然后在你的hexo根目录下右键Git Bash，输入命令，git config –global user.name ‘这里面输入你git账号名称’
然后回车，回车后再输入git config –global user.name ，输完后回车， 查看你的用户名是否设置成功。
第二步，输入 git config –global user.email ‘这里输入你的邮箱’ 然后回车，
回车后输入 git config –global user.email 再回车 ，查看你的邮箱是否设置成功。
（2）创建密钥，输入 ssh-keygen -t rsa -C ‘这里输入你的邮箱’ ，然后回车。回车后，
就会在你的系统里面创建密钥。后面会显示出一个文件目录地址，依照这个路径打开文件目录，我的是/c/Users/MECHREVO/.ssh/id_rsa ，就是在C盘下的Users文件夹，
但是我这个电脑不太一样，我电脑显示的是中文用户两个字，所以这个要注意一下，剩下的依照目录找下去，
直到找到和id_rsa这个文件同级目录下的id_rsa.pub文件，然后用代码编辑器打开，里面就是你的密钥，
全选复制，然后去你的github，点击右上角头像下面的小箭头，然后进入设置，从下往上数，第二个，
github一般显示的是英文，你可以在页面空白处右键，然后点击翻译成中文，进入设置后，点击SSH和GPG，
从上往下数第六个，进来后，点击新的SSH密钥，然后标题就输入你的github名字就好，下面的键里面就把你刚刚
复制的密钥粘贴进来。最后点击添加SSH密钥，到现在，添加密钥成功。
还有一种情况就是，你已经有密钥了。那你输入ssh-keygen -t rsa -C ‘这里输入你的邮箱’ ，这个后，显示出那个
文件的目录就 ctrl + c 停止当前的操作，然后也是和上面一样把密钥粘贴到你的新github上面，我是创了一个新的
github账号，所以才要重新粘贴一次密钥，如果是用旧的git账号就不用这样做。
（3）粘贴完密钥后，输入一行命令，ssh -T git@github.com ，如果直接显示Hi （你的git账号名字）! You’ve successfully authenticated, but GitHub does not provide shell access. 就表示成功了。如果显示出来的有一行这样的英文Are you sure you want to continue connecting (yes/no)? 那你就在后面输入 yes 就完成了，是问你是否要和github实行远程连接。
（4）用编辑器打开你的hexo根目录下的_config.yml文件，找到最下面那行type: 然后空格一下再输入 git，type的冒号后面和输入的内容要有一个空格，不然会没效果。然后回车，再下面再新加一行代码，repo: 冒号后面也需要一个空格，然后去到你的github上找到你的那个项目，点进去后会看到快速设置，看到HTTPS和SSH，点击SSH，然后把后面的地址复制下来，粘贴到repo的后面。再回车，输入branch: master
(5)
